# Drive DJ 버전 노트

> 버전은 **배포(GitHub Pages) 기준**으로 1씩 증가한다. 로컬 수정만으로는 버전을 올리지 않으며, 실제 배포할 때만 버전업 + 이 파일에 기록한다. 배포 규칙·현황은 [[HANDOFF]] 참조.

## v31 — 2026-07-01
- **친구 배포용**: Client ID를 코드에 내장(dj-bot), Client ID 입력창 제거 → 유저는 "Spotify로 로그인"만. 안내문구 친구용(Premium + 활성 기기).
- (배포 전제: Spotify 대시보드 User Management에 각 친구 이메일 등록 ≤25명, redirect URI = Pages URL.)

## v30 — 2026-07-01
- **장르 필터 UI/로직 제거** (Spotify 장르 거의 비어 무력 + 낭비 호출 제거). 장르는 백엔드가 Last.fm으로 계속 수집.
- **DB 추천 파운데이션 배선**: 서빙/피드백을 백엔드 `log` 함수로 집계 기록.
  - `users`(유저 마스터, 로그인 시 upsert), `situation_track_stats`(상황×곡 성과), `user_situation_prefs`(유저×상황 평균BPM·장르/아티스트 가중치). 곡당 폭증 없는 집계 방식.
  - 상황에 **activity 차원**(현재 driving 고정; 러닝/사이클링/커뮤팅은 향후 디바이스 감지).
- **UI 단순화**: 👎 싫어요·⏭ 다음곡 버튼 제거. 좋아요/↻/취향목록만. 스킵·완청은 **재생 진행도로 자동 판정**(끝12초=완청+1.0 / 10초내=−1.0+제외 / 60초내=−0.5 / 그외=−0.3) → 점수·DB 반영.

## v29 — 2026-06-23
- **곡 속성 수집 시작(Phase 2)**: 추천된 곡을 백엔드 `enrich` 함수로 보내 BPM·energy·valence·danceability·acousticness를 모아 `track_facts`에 캐싱.
- 출처: ReccoBeats(audio-features, ID기반·무료급) 주력 + Deezer(bpm 폴백). 한 번 분석된 곡은 재사용.
- 재생은 안 막고 백그라운드로 수집. 아직 추천 로직엔 미반영(데이터 축적 단계).

## v28 — 2026-06-23
- **키 보호 백엔드 도입(Phase 1 시작)**: Supabase 프로젝트 `dj-bot` + Edge Function `yt-search`.
- YouTube 검색을 백엔드가 키를 들고 대신 호출 → **클라이언트에서 YouTube API 키 제거**(입력칸 삭제). 키는 Supabase Secrets에만.
- 웹은 Spotify 로그인만으로 동작. (anon 키는 공개용이라 클라에 둠.)

## v27 — 2026-06-22
- 장르 필터를 **검색형 드롭다운(datalist) + 선택 칩** 방식으로 변경 (직접 타이핑 → 오매칭 방지).
- 장르 목록은 **Spotify 실제 장르 기준**: 기본 흔한 장르로 시작 + 추천 때마다 발견된 장르를 자동 누적(localStorage).
- 포함/제외 각각 다중 선택, 칩 ✕로 제거.

## v26 — 2026-06-22
- **취향 시스템 재설계**: 선호 단위를 "아티스트 + 장르" **결합 카테고리**로 점수화 (예: `아이유 / k-pop`).
- 점수 규칙: 완청 +1.0 / 👍 좋아요 → 별도 리스트 + +1.0 / 👎·다음곡 스킵 = 곡 제외 + 감점(10초내 -1.0, 1분내 -0.5, 그 외 -0.3).
- 취향 패널 편집: 선호 카테고리 [삭제], 좋아요·제외 곡 [삭제/해제], **곡 클릭 → 다음 재생 큐에 추가**.
- 추천 정렬에 선호 점수 반영(+다양성 랜덤).
- 한국/해외 비율 슬라이더 제거 → 한국어·영어 검색 절반씩 자동 혼합.
- 장르 필터(포함/제외, 부분일치) 추가 — Spotify 아티스트 장르로 곡 거름.

## v25 이전 — 2026-06-16 ~ 06-21 (요약)
- **YouTube + Spotify 하이브리드**: YouTube 플레이리스트 채굴로 곡 제목 발굴 → Spotify에서 정확매칭(유사곡 제외) → Spotify 백그라운드 재생.
- 이유: 주행 중 네비가 포그라운드여야 하므로 임베드 YouTube(백그라운드 불가) 대신 Spotify 원격 제어. Spotify 플레이리스트 곡 접근은 403으로 막혀 YouTube를 발굴원으로 사용.
- Spotify OAuth(PKCE) 클라이언트 인증, 곡별 좋아요/싫어요 취향 학습, 위치·날씨·시간·속도 기반 무드 매핑.
- 그 이전: YouTube IFrame 기반 → 백그라운드 재생 한계로 Spotify 전환.
